# Contributing to AI RPI Protocol

Thank you for your interest in contributing to the AI RPI Protocol framework!

## Development Process

All contributions should follow the RPI (Research, Plan, Implement) methodology:

### 1. Research Phase
- Understand the problem thoroughly
- Review existing code and patterns
- Identify constraints and dependencies
- Use the `templates/research.md` template

### 2. Plan Phase
- Define clear objectives
- Break down into manageable tasks
- Identify success criteria and risks
- Use the `templates/plan.md` template

### 3. Implement Phase
- Make minimal, focused changes
- Write tests for new functionality
- Update documentation
- Follow the `templates/implement.md` checklist

## Pull Request Guidelines

- Keep PRs small and focused
- Follow the RPI phases for all changes
- Include appropriate tests
- Update documentation
- Pass all security scans

## Code of Conduct

- Be respectful and constructive
- Focus on the problem, not the person
- Welcome diverse perspectives
