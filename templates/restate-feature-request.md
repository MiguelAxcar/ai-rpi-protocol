# Restate Feature Request

**When to use:** User asks for a new feature, enhancement, or capability. Use BEFORE Phase R on non-trivial requests.

## Format

**Request:** [One sentence describing what the user wants to achieve]

**User story** (if applicable): As a [role], I want [goal], so that [benefit].

**In scope:**
- [item 1]
- [item 2]

**Out of scope:**
- [item 1]

**Unclear (need to clarify):**
1. [question 1]
2. [question 2]

**Early trade-offs:**
- [trade-off 1] — why it matters: [impact]
- [trade-off 2] — why it matters: [impact]

**Expected outcome:** After this change, users will be able to [outcome].

**Next:** I'll research [what], plan [what], then implement [what].

Does this match what you want? [yes/no/adjustments]
